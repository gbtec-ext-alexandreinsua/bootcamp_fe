JSON vs Objetos Literales en JavaScript

Introducción

En JavaScript, es común trabajar con datos en dos formatos: JSON (JavaScript Object Notation) y objetos literales. Aunque ambos pueden parecer similares, tienen características y usos específicos.

Explicación de JSON

JSON (JavaScript Object Notation) es un formato de texto ligero utilizado para intercambiar datos entre aplicaciones. JSON es independiente de JavaScript y se puede usar en diferentes lenguajes de programación.

Características de JSON:

  • Es un formato de texto, por lo que se representa como una cadena (string).
  • Sus claves y valores deben estar entre comillas dobles (").
  • Solo soporta datos simples como números, cadenas, true, false, null, arrays y objetos JSON.

Ejemplo de JSON:

{
    "nombre": "Carlos",
    "edad": 30,
    "esEstudiante": false,
    "Aficiones": ["Baloncesto", "Repostería francesa", "Novela fantástica"]
}

Explicación de los objetos literales (literal objects) en JavaScript

Un objeto literal en JavaScript es una estructura de datos que permite agrupar pares clave-valor. Se usa directamente en el código y es nativo de JavaScript.

Características de los objetos literales:

  • Sus claves pueden estar entre comillas o sin comillas.
  • Permiten incluir funciones, conocidas como métodos, que realizan operaciones dentro del objeto.

Ejemplo de un objeto literal:

const persona = {
    nombre: "Carlos",
    edad: 30,
    esEstudiante: false,
    saludar: function() {
        console.log("¡Hola! Mi nombre es " + this.nombre);
    }
};

Métodos en objetos literales

En JavaScript, un método (también) es específicamente una función almacenada como una propiedad de un objeto. Los métodos permiten que un objeto tenga comportamientos específicos y puedan trabajar con los datos internos del objeto utilizando la palabra clave this, que se refiere al propio objeto.

Definir y ejecutar métodos

Para definir un método en un objeto literal, simplemente declaramos una propiedad cuya valor sea una función. Para ejecutar el método, usamos la notación de punto o corchetes, como con cualquier otra propiedad, pero con paréntesis () al final para indicar que estamos ejecutando una función.

Ejemplo de creación y ejecución de métodos:

const coche = {
    marca: "Toyota",
    modelo: "Corolla",
    año: 2020,
    encender: function() {
        console.log("El coche está encendido.");
    },
    detalles: function() {
        console.log("Este coche es un " + this.marca + " " + this.modelo + " del año " + this.año);
    }
};

// Ejecutar los métodos
coche.encender();  // Muestra: "El coche está encendido."
coche.detalles();  // Muestra: "Este coche es un Toyota Corolla del año 2020."

Uso de this en métodos

La palabra clave this dentro de un método se refiere al objeto que contiene el método, lo que permite acceder a otras propiedades dentro del mismo objeto. En el ejemplo anterior, this.marca, this.modelo y this.año permiten acceder a las propiedades de coche desde dentro del método detalles.

Acceso a las propiedades de un objeto literal

En JavaScript, podemos acceder a las propiedades de un objeto literal usando:

  • Notación de punto: objeto.propiedad
  • Notación de corchetes: objeto["propiedad"]

Ejemplo de acceso a propiedades:

const persona = {
    nombre: "Carlos",
    edad: 30
};

// Acceso usando notación de punto
console.log(persona.nombre); // "Carlos"

// Acceso usando notación de corchetes
console.log(persona["edad"]); // 30

Ejemplos

Ejemplo 1: Método que muestra mensaje con mombre

const estudiante = {
    nombre: "Sofía",
    saludar: function() {
        console.log("¡Hola! Soy " + this.nombre);
    }
};

// Ejecutar el método
estudiante.saludar();  // Muestra: "¡Hola! Soy Sofía"

Ejemplo 2: Método con cálculo basado en propiedades

const circulo = {
    radio: 5,
    calcularArea: function() {
        return Math.PI * Math.pow(this.radio, 2);
    }
};

// Ejecutar el método
console.log("Área del círculo:", circulo.calcularArea()); // Muestra el área calculada

Comparación de objetos

En JavaScript, dos objetos con el mismo contenido no son iguales si no son la misma referencia en memoria.

Ejemplo: Creamos dos objetos con el mismo contenido y los comparamos usando ===.


const obj1 = { nombre: "Juan", edad: 30 };
const obj2 = { nombre: "Juan", edad: 30 };
console.log("¿Son iguales obj1 y obj2?:", obj1 === obj2); // Resultado esperado: false

El resultado es false porque obj1 y obj2 son dos referencias distintas en memoria.

Creación de objetos usando una función factoría

Una función factoría es una función que crea y retorna un objeto, permitiéndonos crear múltiples objetos con la misma estructura de forma sencilla.

Ejemplo: Creamos una función crearPersona para generar objetos de tipo "persona".


    function crearCirculo(radio) {
      return {
        radio: radio,
        calcularArea: function () {
          return Math.PI * Math.pow(this.radio, 2);
        },
      };
    }

const c1 = crearCirculo(5);
const c2 = crearCirculo(10);
console.log("Círculo 1:", c1.calcularArea()); // Circulo 1: 78.53981633974483
console.log("Círculo 2:", c2.calcularArea()); // Circulo 2: 314.1592653589793

La función factoría crearPersona nos permite crear múltiples objetos "persona" con diferentes valores para nombre y edad.

Uso de JSON.stringify y JSON.parse para copia profunda y comparación de objetos

Los métodos JSON.stringify y JSON.parse son útiles para:

  • Comparar el contenido de dos objetos de manera efectiva.
  • Crear una "copia profunda" de un objeto (es decir, una copia sin referencias compartidas).

Ejemplo de comparación: Convertimos dos objetos en cadenas JSON y luego las comparamos.


const obj3 = { nombre: "Juan", edad: 30 };
const obj4 = JSON.parse(JSON.stringify(obj3)); // Copia profunda
console.log("¿Son iguales obj3 y obj4?", JSON.stringify(obj3) === JSON.stringify(obj4)); // Resultado: true

En este caso, la comparación da true porque estamos comparando las cadenas JSON generadas, que son iguales.

Ejemplo de copia profunda: Modificamos la copia y observamos que no afecta al objeto original.


const obj5 = { nombre: "Carlos", edad: 22 };
const obj5Copia = JSON.parse(JSON.stringify(obj5));
obj5Copia.edad = 23;
console.log("Original:", obj5); // Resultado: { nombre: "Carlos", edad: 22 }
console.log("Copia Modificada:", obj5Copia); // Resultado: { nombre: "Carlos", edad: 23 }

La copia obj5Copia es independiente de obj5, por lo que los cambios en la copia no afectan al original.
