Flujo de trabajo para resolver issues

3. reparto por parejas las tareas
4. cada pareja crea una rama especifica por tarea, la sube a repo y crea la PR, pero NO la mergea
        número issue + nombre archivo
        12/hola-mundo
5. subir rama
6. crear PR y dejarla abierta 
        
Revisar PR
1. Descargar rama: pull main > checkout rama 
2. Comprobar código
3. Comprobar funcionamiento 
4a. Si funciona se hace el merge y se anuncia por el canal de teams de que se hizo el merge     
        En este momento TODOS deben actualizar su repo local.
                a. checkout a main, git pull, checkout a rama, git merge main 
4b. Si non funciona, hay que agregar un comentario e avisar con un privado al asignado
