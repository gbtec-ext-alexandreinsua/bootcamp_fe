Variables en CSS

¿Qué son las variables en CSS?
Las variables en CSS, también conocidas como custom properties, son una forma de almacenar valores que puedes reutilizar en tu hoja de estilos. Se definen con un prefijo de doble guion -- y se acceden con la función var().

Definición y uso básico

Las variables se suelen definir dentro de un selector, generalmente en el :root (el selector principal del documento), para que estén disponibles globalmente. 

:root {
            --primary-color: #3498db;
            --secondary-color: #2ecc71;
            --font-size-large: 2rem;
            }

            body {
            background-color: var(--primary-color);
            color: var(--secondary-color);
            font-size: var(--font-size-large);
            }

Ventajas de usar variables en CSS

  • Reutilización: Evitas escribir el mismo valor en múltiples lugares.
  • Mantenibilidad: Cambiar el valor de una variable en un solo lugar actualiza todos los estilos donde se usa.
  • Consistencia: Garantiza que los colores, tamaños y otros valores se mantengan uniformes en todo el diseño.
  • Personalización dinámica: Las variables pueden cambiar según el contexto, como en temas oscuros/claro.

Ejemplo de cambio de tema:

:root {
            --background-color: white;
            --text-color: black;
            }

            body.dark-theme {
            --background-color: black;
            --text-color: white;
            }

            body {
            background-color: var(--background-color);
            color: var(--text-color);
            }

Scope (Alcance)

Las variables pueden definirse globalmente (en :root) o localmente (en cualquier selector). Las variables locales solo son accesibles dentro del selector donde se definieron. 

:root {
            --global-color: #e74c3c;
            }

            .container {
            --local-color: #8e44ad;
            color: var(--local-color);
            }

            .footer {
            color: var(--global-color);
            }

¿Por qué usar :root y no html?

¿Por qué se usa :root y no html?
El selector :root se utiliza en lugar de html porque está diseñado específicamente para trabajar con las custom properties (variables) en CSS de manera global y jerárquica.

Razones para usar :root sobre html

  • Especificidad: El selector :root tiene la misma especificidad que el selector html, pero es semánticamente más claro cuando defines variables CSS globales.
  • Herencia y Modificaciones: Al usar :root, puedes redefinir las variables en otros contextos o scopes sin conflictos. Por ejemplo: 
    :root {
                    --main-color: blue;
                    }

                    body.dark-mode {
                    --main-color: black;
                    }
  • Compatibilidad con la especificación CSS: :root es parte de la especificación CSS para representar la raíz del árbol del documento, lo que lo hace más explícito en el contexto de definir estilos que se aplican a todo.
  • Claridad semántica: Usar :root indica directamente que las propiedades definidas tienen alcance global en el documento, lo cual es más intuitivo al trabajar con variables.

¿Se puede usar html en lugar de :root?

Sí, técnicamente puedes usar el selector html para definir variables. Sin embargo, :root es preferido porque:

  • Está diseñado específicamente para el propósito de trabajar con la raíz.
  • Mejora la claridad y el propósito del código.

Fallos y valores por defecto

Si una variable no está definida o no es válida, puedes especificar un valor por defecto en var(). 

h1 {
            color: var(--non-existent-variable, #000); /* Usará #000 como fallback */
            }
