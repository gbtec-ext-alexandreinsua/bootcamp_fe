CSS Anidado (Nesting CSS)

CSS Anidado: El CSS anidado es una funcionalidad más o menos reciente. Esta funcionalidad permite estructurar reglas CSS dentro de otras, mejorando la legibilidad y organización.

CSS anidado nativo

La funcionalidad de anidamiento ya se está integrando en el CSS estándar, inspirándose en preprocesadores como SASS. En el CSS nativo, se utiliza el símbolo & para referirse al selector padre. Esto mejora la claridad y elimina redundancias en el código.

Ejemplo de CSS Anidado Nativo:

.container {
            padding: 20px;
            background-color: #f4f4f4;

            & h1 {
            color: #333;
            }

            & p {
            font-size: 16px;
            }

            & .button {
            background-color: #3498db;
            color: white;
            }
            }

Ventajas del CSS Anidado Nativo

  • Elimina la necesidad de preprocesadores: Puedes escribir CSS más compacto y claro directamente en el navegador.
  • Reduce la repetición: Agrupa las reglas de estilo relacionadas sin duplicar selectores.
  • Estandarización: Los navegadores están implementando esta característica como parte del estándar CSS.

Sintaxis del CSS anidado

Existen dos sintaxis diferentes para anidar el css.

Sintaxis simple (sin el selector de anidación &)

Cuando no usas el signo &, el selector dentro del bloque anidado se comporta como un nuevo selector, independiente del contexto del padre. 

.button {
            color: red;

            .icon {
            font-size: 20px;
            }
            }

Aquí, .icon se refiere a un selector totalmente independiente de .button. Como resultado, en CSS compilado se generará algo como: 

.button {
            color: red;
            }

            .icon {
            font-size: 20px;
            }

Sintaxis avanzada (con el selector de anidación &)

Cuando se usa el signo &, el selector dentro del bloque anidado se comporta como un selector hijo , dependiente del contexto del padre. 

.button {
            color: red;

            & .icon {
            font-size: 20px;
            }
            }

Aquí, .icon se refiere a un selector dependiente de .button. Como resultado, en CSS compilado se generará algo como: 

.button {
            color: red;
            }

            .button  .icon {
            font-size: 20px;
            }

Uso Condicional:

Puedes usar herramientas como PostCSS para transpilar CSS anidado a una sintaxis estándar si necesitas compatibilidad con navegadores más antiguos.

Reglas y Limitaciones

  • No se recomienda anidar en exceso: Aunque el anidamiento mejora la claridad, niveles excesivos pueden dificultar el mantenimiento.
  • Jerarquía clara: Los selectores deben seguir una jerarquía lógica que refleje la estructura del DOM.

Ejemplo Complejo:

.navbar {
            background-color: #222;

            & ul {
            list-style: none;
            margin: 0;
            padding: 0;

            & li {
            display: inline-block;
            padding: 10px;

            & a {
            text-decoration: none;
            color: white;

            &:hover {
            color: #3498db;
            }
            }
            }
            }
            }
