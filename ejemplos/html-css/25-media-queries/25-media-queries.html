Media queries

¿Qué son la media queries?

Un Media Query es una regla en CSS que permite aplicar estilos de manera condicional según ciertas características del dispositivo o la ventana de visualización (viewport), como el tamaño de la pantalla, la orientación, o la resolución. Se utiliza para crear diseños web adaptativos, también conocidos como diseños responsivos, que se ajustan automáticamente al dispositivo del usuario (móviles, tabletas, dispostivos de escritorio, etc.).

Permiten que una página web se vea bien y funcione correctamente en cualquier dispositivo sin necesidad de crear múltiples versiones.

Es una de las piezas del Responsive Web Design.

Una media query esta compuesta por

  • Condición: Se evalúa una característica del dispositivo, como el ancho de la pantalla (max-width: 600px).
  • Estilos condicionales: Si se cumple la condición, los estilos dentro de la Media Query se aplican.

Sintaxis general de las media queries

La sintaxis general de una media querya es la siguient

@media [media-type] (condición) {
    /* Reglas CSS que se aplican si se cumple la condición */
}

donde

  • @media: Es la palabra clave que inicia la Media Query.
  • media-type: especifica el tipo de medio para el cual se aplicarán estilos. Los valores pueden ser screen para pantalla, print para medios de impresion o all, que es el valor por defecto.
  • (condición): Define las características del dispositivo o ventana (por ejemplo, ancho, altura, orientación). Normalmente se comprueba en ancho de la pantalla con la propiedad width, pero también es frecuente usar otras como orientation o resolution.
  • Reglas CSS: Se colocan dentro de las llaves {} y se aplican solo si la condición es verdadera.

Ejemplo básico

Este es un ejemplo de media query que cambia el fondo del div dependiendo del tamaño de la pantalla 

.ex1 {
        width: 350px;
        height: 150px;
        padding: 1rem;
        color: #eee;
        text-align: center;
        background-color: #333;
        border-radius: 1rem;
        display: flex;
        align-items: center;
    }
    
    @media (width > 900px) {
        .ex1 {
            color: #333;
            background-color: #eee;
            font-weight: bolder;
        }
    }
Este div cambia de fondo dependiendo del ancho de la ventana

Variaciones de sintaxis sobre el ancho de la pantalla

Inicialmente se usaban las propiedas max-width y min-width para establecer valores máximos y mínimos en las condiciones de la media query. A partir del 2021 tenemos disponible la sintaxis de rango con la propiedad width.

Esta sintaxis permite manjer operadores de comparación como , <, >=, o <= que hacen más legible la regla

Por ejemplo, una regla que se debe aplicar cuando en ancho del viewport esta comprendida entre 500px y 900px puede ser escrita de dos formas: 

.ex2 {
    padding: 1rem;
    font-weight: bolder;
    text-align: center;
}

/* Esta es la sintaxis moderna */ 
@media (500px <=width <=900px) {
    .ex2 {
        color: #fee;
        background-color: #333;
    }
} 
        
/* Esta es la sintaxis tradicional */
    @media (min-width:500px) and (max-width:900px) {
        .ex2 {
            color: #fee;
            background-color: #333;
        }
}

Aplicación de las media query al responsive design

En el contexto de Responsive Web Design, las media queries se utilizan para crear reglas de estilo específicas para diferentes tamaños de pantalla. Esto significa que una página web puede tener un diseño fluido que cambia de acuerdo con el tamaño y las características del dispositivo que la está visualizando.

Por ejemplo, supongamos que tenemos una vista definida para mostrar varios elementos en una línea. Si la pantalla es pequeña, solo se mostrará un elemento, mientras que si la pantalla es grande, se mostrarán todos los elementos.

¿Cómo se aplica la media query? ¿Se usan para especificar en comportamiento en dispositivos móviles?. No

Nosotros debemos seguir el principio de diseño adaptativo llamado Mobile First que establece que primero se debe diseñar para dispositivos móviles, y luego para otros dispositivos. Por tanto el punto de partida es la vista para los dispositivos móviles, después para tablets y finalmente para grandes pantasllas de escritorio

Por tanto, se aplican las media queries para definir el comportamiento en tamaños de pantalla medios, grandes y muy grandes.

Puntos de ruptura

Los puntos de ruptura son los valores de ancho en los que el diseño de una página web cambia para adaptarse a diferentes tamaños de pantalla. Los valores más comunes están basados en las resoluciones de los dispositivos más populares. La tabla a continuación incluye la equivalencia de píxeles (px) a em (suponiendo que la base del tamaño de la fuente es de 16px), lo que permite hacer que los breakpoints sean más escalables.

Tabla de Puntos de Ruptura Comunes

Dispositivo Ancho (en píxeles) Equivalencia en em
Móvil pequeño < 480px < 30em
Móvil grande 480px - 600px 30em - 37.5em
Tableta en orientación vertical 601px - 768px 37.56em - 48em
Tableta en orientación horizontal 769px - 1024px 48.06em - 64em
Escritorio pequeño 1025px - 1200px 64.06em - 75em
Escritorio grande 1201px - 1600px 75.06em - 100em
Pantalla extra grande > 1600px > 100em

Explicación de los puntos de ruptura más comunes:

1. Móviles (pantallas pequeñas):

max-width: 480px o max-width: 30em: Este es un punto de ruptura común para móviles pequeños. El diseño debe adaptarse a pantallas pequeñas, como los smartphones más antiguos o de gama baja.

2. Móviles grandes:

max-width: 600px o max-width: 37.5em: Para teléfonos móviles más grandes. Este es un breakpoint común para los teléfonos modernos.

3. Tabletas en orientación vertical:

max-width: 768px o max-width: 48em: Este valor se usa generalmente para tabletas en modo vertical (retrato). También es común usar un rango que cubra de 600px a 768px.

4. Tabletas en orientación horizontal:

max-width: 1024px o max-width: 64em: Para tabletas en modo horizontal (paisaje). Los dispositivos como el iPad en orientación horizontal suelen tener esta resolución.

5. Escritorios pequeños:

min-width: 1025px o min-width: 64.06em: Para pantallas de escritorio de tamaño más pequeño o pantallas de laptops con resolución de 1024px o más.

6. Escritorios grandes:

min-width: 1201px o min-width: 75.06em: Este punto de ruptura cubre pantallas de escritorio de gama alta o monitores grandes.

7. Pantallas extra grandes:

min-width: 1600px o min-width: 100em: Para pantallas de muy alta resolución o monitores ultra anchos.

Prácticas comunes al definir puntos de ruptura:

  • Usar unidades relativas como em o rem: Usar em en lugar de px permite que los breakpoints se ajusten automáticamente si el tamaño de la fuente base cambia. Es útil cuando se usa un enfoque de Mobile First o para hacer que el diseño sea más fluido. 
    @media (with <= 48em) {
    /* Estilos para pantallas pequeñas */}
  • Mobile First: Se recomienda usar el enfoque de Mobile First: comenzar con el diseño para móviles y luego usar las Media Queries para pantallas más grandes. Esto hace que el diseño sea más accesible y eficiente, ya que los estilos por defecto se aplican a pantallas pequeñas, y luego se agregan modificaciones para pantallas más grandes. 
    /* Estilos para móviles */
    .contaiter {
        /* otras reglas */
    }
            
    /* Media Query para pantallas grandes */
        @media (width > 48em) {
            .container {
                flex-direction: row;
              }
        }
  • Utilizar breakpoints fluidos: En lugar de ajustarse estrictamente a valores específicos, muchos diseñadores optan por breakpoints fluidos, que se adaptan dinámicamente en función de los cambios en el diseño y el contenido. Esto es útil en diseños complejos.

Resumen

Los puntos de ruptura (breakpoints) son cruciales en Responsive Web Design para adaptar el contenido a diferentes dispositivos. Los valores más utilizados en píxeles (px) y su equivalencia en em son:

  • 480px o 30em para móviles pequeños,
  • 600px o 37.5em para móviles grandes,
  • 768px o 48em para tabletas en modo vertical,
  • 1024px o 64em para tabletas en modo horizontal,
  • 1025px - 1200px o 64em - 75em para escritorios pequeños,
  • 1201px - 1600px o 75em - 100em para escritorios grandes y pantallas extra grandes.

Ejemplo de uso de media queries

En el siguiente ejemplo se muestra un diseño de un formulario de búsqueda que se adapta a diferentes tamaños de pantalla. El tamaño de la pantalla determina el tamaño del campo de búsqueda y el tamaño de la barra de búsqueda.

Para hacer que el diseño sea más accesible, se utiliza el enfoque de Mobile First y se aplica el estilo para pantallas pequeñas y luego se aplica el estilo para pantallas más grandes.

Para hacer que el diseño sea más fluido, se utiliza el enfoque de Breakpoint Fluid y se aplica el estilo para pantallas pequeñas y luego se aplica el estilo para pantallas más grandes.

Los puntos ruptura que se usan son para móviles (hasta 600px) y para tabletas (768px - 1024px) y escritorio (a partir de 1025px)
