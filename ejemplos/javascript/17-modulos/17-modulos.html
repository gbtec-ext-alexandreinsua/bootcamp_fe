Guía de Módulos ES6

¿Qué son los módulos de ES6?

La evolución del desarrollo web ha generado desafíos cada vez más complejos en la gestión del código. En este contexto, los módulos emergen como una solución fundamental para organizar y estructurar aplicaciones JavaScript, respondiendo a una necesidad crítica de modularidad y mantenibilidad.

Un módulo puede entenderse como unaunidad autónoma unidad autónoma de código que encapsula funcionalidades relacionadas. Al igual que un sistema de compartimentos estancos en un barco, cada módulo gestiona sus propios recursos internos, exponiendo únicamente aquellas interfaces necesarias para la interacción externa. Esta aproximación permite un control preciso de la visibilidad y accesibilidad del código.

Los módulos de ES6 permiten dividir tu código JavaScript en archivos más pequeños y reutilizables. Puedes exportar funcionalidades desde un archivo y luego importarlas en otro, mejorando la organización y mantenimiento del código.

Estos módulos forman un estándar para la exportación e importación de funcionalidades, normalizando prácticas que anteriormente dependían de bibliotecas externas o convenciones informales. Las palabras clave export e import se convierten en herramientas que definen los límites y las interacciones entre diferentes componentes de software.

Exportar una funcionalidad

Exportación con nombre


// archivo.js
export const saludo = "Hola, mundo";
export function suma(a, b) {
    return a + b;
}

Exportación por defecto


// archivo.js
export default function saludo() {
    console.log("Hola desde una función exportada por defecto");
}

Importar una funcionalidad

Importación con nombre


// main.js
import { saludo, suma } from './archivo.js';
console.log(saludo);
console.log(suma(3, 4));

Importación por defecto


// main.js
import saludo from './archivo.js';
saludo();

Renombrar una importación


import { suma as calcular } from './archivo.js';
console.log(calcular(5, 6));

Uso con la etiqueta <script>

Ejemplo básico


<script type="module" src="main.js"></script;>

Módulos inline


<script type="module";>
    import { saludo } from './archivo.js';
    console.log(saludo);
</script>

Diferencia entre type="module" y defer

Característica type="module" defer
Soporte para módulos Sí, permite usar import y export. No, no admite módulos ES6.
Ejecución diferida Sí, como defer, el script se ejecuta después de cargar el DOM. Sí, el script se ejecuta tras cargar el DOM.
Aislamiento Sí, cada módulo tiene su propio ámbito (scope). No, comparte el ámbito global con otros scripts.
Relaciones entre scripts Admite dependencias explícitas mediante import. No gestiona dependencias.
Compatibilidad Sólo navegadores modernos que soporten ES6. Amplia compatibilidad, incluido soporte antiguo.

Notas importantes

  • Los módulos se cargan de forma diferida automáticamente. No necesitas añadir defer a un <script type="module">, ya que los navegadores lo manejan así por defecto.
  • Cuando se usa type="module", los navegadores exigen que los archivos se sirvan desde un servidor web y respetan las políticas de CORS.
